# 星藏家代码审查记录

审查日期：2026-07-14
目标版本：`0.9.8`

## 审查范围

本轮不是针对单一缺陷的同类排查，而是对整个工程的所有源码与主要功能状态机进行审查。覆盖：

- Electron 主窗口、Preload、IPC、WebView 登录、窗口/外链/剪贴板边界；
- Bilibili API、Cookie/头像、收藏夹读取与事务同步；
- SQLite 持久化、Workspace、任务领取/租约/中止/提交/崩溃恢复；
- 外部 Agent API、Worker/`workId` 鉴权、工具轮询和错误指令；
- 内置 Agent、单视频模式、模型失效、上下文整理和流式生命周期；
- ToolRunner、资源池、下载/FFmpeg、常驻 GPU/CPU ASR、失败分类；
- 视频缓存队列、视频库、封面、删除和启动恢复；
- RAG 供应商、上下文压缩、知识库原文/原图、附件、CMD/浏览器权限；
- Markdown 验收、产物改名/迁移、文档库、导出和 Mermaid；
- 依赖下载/校验/事务安装、便携打包、发布脚本和第三方许可；
- 全部 Renderer 事件、动态 HTML、滚动/展开状态、错误与按钮恢复；
- 全部自动化脚本、源码路径、版本与 Release 门禁。

## 已修复问题

1. **严重：异步 HTTP 异常可逃逸路由错误处理。** Agent API 现在拥有统一 Promise 错误边界，格式错误或超限请求不会形成未处理拒绝。
2. **严重：本地 API 开放通配 CORS。** 已拒绝无关浏览器 Origin，并移除通配 CORS；JSON 请求体限制为 1 MiB。
3. **高：WebView、主窗口和隐藏浏览器边界不足。** 已启用沙箱、Bilibili 导航白名单、弹窗/下载阻断、私网与 DNS 检查及 CSP。
4. **高：伪装 Bilibili URL、重定向或远程图片可触发非预期网络/渲染。** 视频链接逐跳校验官方域名；Markdown/RAG/封面只接受受管本地图片、可信 Bilibili HTTPS 图片或受限栅格 Data URL。
5. **高：系统加密不可用时存在明文密钥回退风险。** 账号密码和模型 API Key 现在直接拒绝保存；敏感额外请求头也被拒绝。
6. **高：提交产物可借符号链接、越界路径或伪造图片逃逸。** 验收限定分配目录、普通文件、深度/数量/大小和真实栅格签名，并要求临时媒体已清理。
7. **高：sql.js 数据库直接覆盖写入。** 改为临时文件、`fsync`、备份和启动恢复；事务保存失败会恢复提交前数据库。
8. **高：任务中止、崩溃或旧 `workId` 仍可能续做。** 所有中止入口统一取消工具、清除尝试文件、回退 `pending`、使一次性 `workId` 失效；旧调用收到 `WORK_ATTEMPT_ENDED`。
9. **高：失效视频被当作普通失败反复派发。** 明确删除/下架信号会移除任务、清理缓存、写 `unavailableTasks` 墓碑并禁止同步复活。
10. **高：收藏夹同步可能与 Agent 并发修改库存。** 同步先停止整个收藏夹工作流和活跃尝试，再完整拉取分页并事务提交；中断使用持久化日志回滚，已完成产物独立保留。
11. **高：ASR 纯文本缺少逐句真实时间。** 常驻 faster-whisper 开启词级时间戳并对完整音频连续解码，按标点/停顿/长度组合句子；SRT、时间轴 TXT 和 JSON 必须时间一致才能验收。
12. **高：ASR 请求错误与服务/显卡故障混为一类。** 单视频转写错误只结束该任务；连续原生启动故障才暂停 Worker、回滚任务并输出结构化问题与可能原因。
13. **高：GPU 正忙而 CPU 致命失败时队列会被误拒绝。** 只有所有启用通道均空闲且致命失败时才拒绝；存在忙碌健康通道时继续排队。
14. **高：视频缓存崩溃恢复可能同时由通用工具恢复器和缓存队列启动。** 通用恢复器现在取消缓存管理器名下的旧 Run，由持久化下载 Job 单独从头排队，避免重复进程。
15. **高：RAG 可用摘要索引冒充原始文档。** 新增原始 Markdown 分页读取、发布日期/收藏日期、已移出状态、原图多模态读取和 24 次工具预算。
16. **高：RAG/视频 Agent 上下文接近上限时只能裁剪。** RAG 在 75% 自动压缩长期会话；视频任务仅在 82% 极端路径使用相同供应商/模型的独立 Map/Reduce 请求完整整理当前素材，不跨视频累积上下文。
17. **中：流式刷新反复替换 DOM，造成点击丢失、滚动回顶和展开项关闭。** Agent 页面使用稳定节点增量更新；状态页按稳定键恢复展开和滚动，刷新期间的新完成事件会触发补读。
18. **中：立即停止可重复触发并卡在 stopping。** 前端按会话动作去重，后端停止幂等执行工具取消、任务回滚和 Worker 暂停。
19. **中：单视频完成后文档库不能立即出现。** 内部完成事件会使共享任务快照失效；修订计数保证完成事件与在途读取竞态时再次刷新。
20. **中：重新登录后头像可能被混合内容、Referrer 或 Cookie 策略阻断。** 头像通过 Electron Bilibili Session 下载、校验栅格签名、转为受限 Data URL 并随用户持久化。
21. **中：文档库封面只有远程回退且存在未校验 Renderer 回退。** 主进程依次解析本地封面、`info.json`、常见文件、首帧和可信远程封面；Renderer 只使用该已解析结果。
22. **中：Windows 长文件名和最终目录改名可导致归档失败或缓存索引失效。** 叶名称按完整父路径预算缩短并加哈希；移动有重试/复制兜底，封面和缓存路径随最终目录一起迁移，提交事务可恢复。
23. **中：依赖安装中断可能留下新旧运行时混合状态。** 安装改为 staging、backup 和持久化 journal；启动自动回滚，损坏 journal 隔离后继续启动并在设置页提示。
24. **中：供应商 Base URL、额外请求头和请求时限缺少一致约束。** 公网 HTTP 被拒绝，凭据不允许写入自由请求头，模型请求拥有统一超时与基础设施错误分类。
25. **中：图片复制只信 MIME/扩展名。** 本地、远程和 Data URL 均受路径/私网/大小/重定向/栅格签名检查，SVG 与 MIME 伪装会被拒绝。
26. **中：后台 GPU、租约、依赖和隐藏浏览器定时任务可产生未处理异常。** 定时器和异步回调均增加错误边界，退出时清除审批/浏览器超时并处理 Renderer 加载失败。
27. **中：视频库横竖屏、封面和 BV 交互不完整。** 本地封面与尺寸持久化，播放器按真实比例布局；BV 左键打开原视频、右键复制并提示。
28. **质量：模型漏掉 Mermaid、评论或字幕比较时整单失败。** 内部 Agent 基于真实素材规范化开头、思维导图和评论栏目；验收同时检查顺序、时间轴、字幕比较和 ASR 三份时间数据。
29. **维护性：API、同步、归档、网络、媒体错误和恢复逻辑耦合。** 已拆成独立 Core 模块，并以服务接口连接；内置 Agent 不再反向依赖 HTTP API Server。
30. **发布风险：校验脚本未覆盖全部文档、Python、GPU 模型和版本漂移。** 发布门禁现在扫描全部根文档/包文件，核对 lockfile 版本，编译 JS/Python，运行全套集成测试、`small`/`medium` 真实 GPU 常驻服务和 `npm audit`。

## 保留风险

- 本地 Agent API 没有面向不可信本机进程的强认证，Worker ID 不是互联网安全令牌；不得映射到局域网或公网。
- RAG “完全访问”允许模型以当前桌面用户权限访问沙盒外文件并执行命令，只应给可信供应商和明确任务使用。
- Bilibili Cookie 必须以明文 Netscape 格式供 yt-dlp 使用，应保护 Workspace，不要同步到公开网盘或 Git。
- sql.js 仍采用整库导出，超大数据库的写入成本会增长；规模继续扩大时应评估原生 SQLite/WAL。
- Bilibili 官方接口、页面登录结构和 yt-dlp 提取器可能变化；现有重试、限速和错误分类不能保证第三方服务永久稳定。
- 视频不可用分类只接受明确错误码/文本，未知错误继续按普通失败处理，以免将临时网络故障误删为终态。
- 站内字幕已按所有分 P 枚举，但当前合轨下载/ASR 对多 P 视频尚无专门的“全分 P 拼接并校正累计时间轴”回归样本；发布前把多 P 当作需人工抽查的兼容性边界，后续应以真实公开样本增加自动化。
- Release 尚未代码签名，Windows 可能显示未知发布者提示。

## 验证入口

`npm run verify:release` 是总门禁，内部包含语法、全功能集成、安全、持久化、同步、调度、RAG、内置 Agent、缓存、ASR 格式、真实双模型 GPU 服务和依赖审计。也可单独运行 `package.json` 中每个 `test:*` 脚本定位失败。
