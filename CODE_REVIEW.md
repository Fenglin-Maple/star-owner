# 星藏家代码审查记录

审查日期：2026-07-16

目标版本：`1.0.0`

## 审查范围

本轮覆盖整个工程与主要状态机，不限于单一缺陷：

- Electron 主窗口、Preload、IPC、WebView、外链、剪贴板和 Renderer 事件；
- Bilibili API、Cookie、头像、收藏夹事务同步与回滚；
- SQLite、Workspace、任务领取、中止、崩溃恢复与文档删除；
- 应用内队列 Agent、单视频模式、模型失效和上下文整理；
- ToolRunner、资源池、下载、FFmpeg、GPU/CPU ASR 与失败分类；
- 外部知识库 HTTP API、目录、原文、搜索、图片和路径边界；
- RAG 供应商、上下文压缩、原文/原图、附件、CMD 和浏览器权限；
- 视频缓存、视频库、封面、播放器和删除；
- 依赖安装、便携打包、文档、版本、测试和发布门禁。

## 本版本修复

1. **严重：已经关闭外部视频接单，但旧服务仍保留可达任务代码。** `api-server.js` 已缩减为只读知识库服务器。原 Worker、claim、工具、提交和中止路径在路由入口统一返回 HTTP `410 / EXTERNAL_VIDEO_WORKFLOW_DISABLED`，不存在可绕过的旧处理分支。
2. **高：外部知识库只能看索引，缺少完整原文与原图协议。** 新增协议 `3.0`，提供全量完成文档目录、精确元数据、1 基行号 Markdown 分页、SHA-256、受限搜索、图片列表和二进制读取。
3. **高：知识 API 可能通过数据库路径读取未托管文件。** 文档与图片必须位于已注册 Workspace 的真实路径下；拒绝路径穿越、符号链接逃逸、无效普通文件、超限文档和伪造图片。
4. **高：错误可能泄露本机绝对路径。** 文件竞态、权限、缺失与边界错误统一转换为稳定公开错误码，不返回 SQLite 中的路径或底层文件系统错误文本。
5. **中：搜索可能无限扫描并阻塞。** 单请求限制 128 MiB，返回候选数、实际扫描数、跳过数和 `partial`；搜索摘要明确不是原文。
6. **中：日期和原文分页输入被静默纠正。** 无效日期、倒置范围、非法搜索词和超出文档的起始行现在返回明确 `400/416`，不再悄悄得到误导结果。
7. **高：单视频覆盖仍保存历史修订。** 同收藏夹/BV 只保留一条任务和一个产物；覆盖清理旧产物并原位从头生成，不创建 revision 链或 RAG 旧版本。
8. **高：删除单视频产物后可能回到待派发或再次触发旧产物提醒。** 删除会移除同 BV 单视频任务、关联会话和文件；下次作为全新任务创建。
9. **高：内置收藏夹文档删除可能错误回待派发。** 仅仍在 B 站收藏夹的完成任务恢复为 `pending`。移出收藏夹、原收藏夹删除、单视频和其它本地产物均不恢复。
10. **中：任务总览开关描述与实际内部领取边界不一致。** 外部激活按钮与 IPC 已删除，页面明确说明开关控制应用内工作流；集成测试验证内部 Agent 跳过 `enabled=false`。
11. **中：设置导航只有 HTML 层级，没有三级状态。** 新增三级菜单展开、活动项、持久化和父级联动，Agent 工作列表、工具模块、工具状态位于“设置 -> 状态查询”。
12. **高：仅检测显卡名称无法判断本地 ASR 可运行。** 新检测同时验证项目 Python、faster-whisper、CTranslate2、模型、CUDA 设备、显存、系统内存、架构和线程数，并将结果用于 UI、资源通道和工作流启动拦截。
13. **中：切换 ASR 模型后 CPU 条件不满足时仍可能尝试启动 CPU 服务。** 配置切换只在重新检测后仍支持且开关有效时启动 CPU 常驻服务。
14. **维护性：外部视频 API 字段和单视频版本模块继续制造错误契约。** 删除 Renderer 激活桥、Store 活动收藏夹方法、同步中的外部分发字段和未使用 `task-versions.js`；工具元数据改为应用内调用标识。
15. **质量：启动页和工具页仍引导使用旧接口。** 启动页现在默认收起完整知识库接入提示词；Agent 工具状态页显示只读 API；工具模块说明只允许应用内调度。
16. **高：无音轨视频被当作 FFmpeg 随机失败并反复派发。** 音频提取现在保留 FFmpeg stderr、规范化 Windows 负退出码，并把 `Output file does not contain any stream` 转换为 `NO_AUDIO_STREAM`；工作流生成空 ASR 诊断后继续使用字幕与关键帧。
17. **高：B 站报告收藏数量与分页可见数量不一致时，整个收藏夹永久无法同步。** 同步现在区分报告数、可见数和可见性差值；部分可见快照合并可见更新但不推断缺席任务已移出，完整快照才执行移出与归档。
18. **严重：通用 `does not exist` 把 Markdown 图片/元数据校验错误误判成视频失效。** 失效分类收紧到明确 B 站终止错误码和视频删除措辞；启动迁移会恢复历史误判任务并删除错误墓碑。
19. **中：模型偶发输出错误章节顺序或 `frames/frame-%03d.jpg` 模板路径。** 提交前确定性重排“小结 -> 思维导图 -> 目录”，并用真实帧文件替换占位符，减少无意义的模型重试。
20. **中：视频工具用 `process.exit(1)` 强制关闭时可能触发 Windows libuv `UV_HANDLE_CLOSING` 断言。** 改为设置标准退出码并让 Node 自然清理句柄；`62012` 同时按明确不可见视频处理。
21. **高：视频标题含 `%` 时，FFmpeg 把产物目录误当图片序列模板。** 抽帧改为 MJPEG 图片流后由应用写入编号文件，不再让元数据路径参与 FFmpeg 模板解析；帧清单同时排除遗留的 `frame-%03d.jpg`。B 站收藏夹、缓存收藏夹和单视频模式使用同一修复路径。
22. **中：带编号或装饰的必需章节标题未被结构修复器识别。** 规范化器现在按标题语义识别“小结、思维导图、目录”，统一改成精确 H2 后再排序和校验。

## 延续有效的安全修复

- WebView、主窗口和隐藏浏览器使用沙箱、域名白名单、弹窗/下载阻断、私网与 DNS 检查。
- 账号密码和模型 API Key 在 `safeStorage` 不可用时拒绝保存。
- SQLite 使用临时文件、fsync、备份和恢复；同步与依赖安装拥有持久化事务日志。
- 任务中止、应用退出、崩溃恢复、模型删除和 ASR 基础设施失败统一清理本次尝试并使 work ID 失效。
- 失效视频使用墓碑阻止未来同步复活。
- ASR 使用逐句真实时间戳，RAG 可读取发布日期、收藏日期、原始 Markdown 和原图。
- RAG 75% 自动压缩；视频 Agent 仅在 82% 或供应商超限时执行完整语义整理。
- 动态页面保留稳定节点、展开状态和滚动位置，避免刷新抢走点击。
- 依赖安装使用 staging、backup、SHA-256 和回滚日志。

## 保留风险

- 本地知识库 API 不认证同机其它进程；只能绑定回环地址，不得映射到局域网或公网。
- RAG“完全访问”允许模型以桌面用户权限访问沙盒外文件和执行命令，只能授予可信供应商和明确任务。
- Bilibili Cookie 因 yt-dlp 兼容需要以明文 Netscape 格式保存，应保护 Workspace。
- sql.js 仍采用整库导出；数据库继续增大时应评估原生 SQLite/WAL。
- Bilibili 接口、登录页面和 yt-dlp 提取器会变化，重试与错误分类不能保证第三方永久稳定。
- 视频不可用只接受明确错误；未知错误继续回滚，以免临时网络故障误删任务。
- 多 P 视频已经枚举站内字幕，但全分 P 合轨与累计时间轴仍需要更多真实样本回归。
- Release 尚未代码签名，Windows 可能显示未知发布者。

## 验证入口

`npm run verify:release` 是总门禁。新增关键测试：

- `npm run test:knowledge-api`：协议、目录、日期、原文分页、搜索、图片、ETag、越界和旧接口关闭；
- `npm run test:hardware`：NVIDIA/CUDA、CPU 回退、模型显存/内存和缺失运行时；
- `npm run test:internal-agent`：内部任务开关、单视频唯一覆盖与删除后新建；
- `npm run test:document-lifecycle`：B 站恢复与本地永久删除矩阵；
- `npm run test:media-edge`：无音轨视频识别、百分号路径下三种任务来源抽帧、空 ASR 诊断、继续处理和工具失败句柄清理；
- `npm run test:collection-sync`：事务回滚、部分可见快照、缺席任务保留和错误墓碑恢复；
- `npm run smoke`：端到端只读 API 和核心应用契约。

总门禁还包含全部既有功能测试、JavaScript/Python 语法、双模型真实 ASR 服务和 `npm audit --audit-level=high`。
